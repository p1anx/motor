# TCP/UART应用 - 页面导航功能

## 功能概述

已成功将同步控制功能移至独立页面，实现了页面导航功能。用户可以在主界面和同步控制页面之间自由切换，提供了更好的用户体验。

## 🎯 新功能特性

### 1. 独立同步控制页面
- **专门页面**: 同步控制功能移至独立页面
- **完整功能**: 包含所有同步配置和控制功能
- **使用说明**: 内置详细的使用指南

### 2. 页面导航系统
- **导航栏**: 顶部导航栏显示应用标题和功能按钮
- **页面切换**: 主界面与同步页面间无缝切换
- **状态指示**: 实时显示同步状态

### 3. 现代化界面设计
- **深色导航栏**: 专业的深色主题设计
- **蓝色主题**: 统一的蓝色色彩方案
- **圆角设计**: 现代化的圆角边框
- **悬停效果**: 按钮悬停动画效果

## 🏗️ 界面结构

```
┌─────────────────────────────────────────────────────────┐
│ 导航栏: 应用标题 + 同步控制按钮 + 状态指示器          │
├─────────────────────────────────────────────────────────┤
│ 主界面: TCP通信区域 + UART通信区域                     │
│         (水平分割布局)                                  │
└─────────────────────────────────────────────────────────┘
```

### 页面切换流程
```
主界面 ←→ 同步控制页面
  ↑           ↓
导航栏 ←→ 返回按钮
```

## 📋 使用方法

### 1. 启动应用
```bash
python tcp_uart_unified_app_main_v2.py
```

### 2. 主界面操作
- **TCP通信**: 配置和连接TCP客户端/服务端
- **UART通信**: 配置和连接串口通信
- **数据收发**: 发送和接收数据
- **实时绘图**: 数据可视化显示

### 3. 同步控制页面
- **进入页面**: 点击导航栏的"同步控制"按钮
- **配置同步**: 设置TCP和UART的同步字符串
- **启用同步**: 勾选"启用同步模式"复选框
- **开始同步**: 点击"开始同步"按钮
- **返回主界面**: 点击"← 返回主界面"按钮

## 🔧 技术实现

### 核心组件
- **QStackedWidget**: 页面堆栈管理
- **SyncControlPage**: 独立同步控制页面类
- **导航栏**: 顶部导航界面
- **状态同步**: 页面间数据共享机制

### 关键方法
- `init_main_page()`: 初始化主页面
- `init_sync_page()`: 初始化同步页面
- `show_main_page()`: 显示主页面
- `show_sync_page()`: 显示同步页面
- `update_sync_status_indicator()`: 更新状态指示器

### 页面管理
```python
# 页面堆栈
self.stacked_widget = QStackedWidget()

# 页面索引
0: 主页面
1: 同步控制页面

# 页面切换
self.stacked_widget.setCurrentIndex(0)  # 主页面
self.stacked_widget.setCurrentIndex(1)  # 同步页面
```

## 📊 状态管理

### 同步状态流程
1. **未启用**: 默认状态，同步功能关闭
2. **已启用**: 同步模式开启，等待配置
3. **进行中**: 同步过程进行中
4. **已完成**: 同步成功，开始正常通信

### 状态同步机制
- **主应用状态**: 核心同步状态管理
- **页面状态**: 同步页面状态显示
- **指示器更新**: 导航栏状态实时更新
- **数据共享**: 页面间数据同步

## 🚀 使用流程

### 基本使用步骤
